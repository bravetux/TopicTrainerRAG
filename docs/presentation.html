Introduction

What is TechTrainer AI?

A production-grade employee training chatbot built on the Strands Agents SDK — enabling employees to access, navigate, and interact with training materials across QA testing and data engineering technologies.

🎯 Purpose

  • Instant Q&A over all training documents
  • Interactive quizzes with scoring and explanations
  • Personalized learning path recommendations
  • AI-generated training module authoring
  • Visual progress dashboard per technology
  • Background document indexing with live progress

🗂 Knowledge Base

Any Topic
Upload documents on any subject — the system indexes and retrieves from whatever topics you add
Any Document Format
PDF · DOCX · PPTX · XLSX · TXT · Markdown
Fully Extensible
Add custom topics at runtime via the Knowledge Base UI — no code changes required

Problem Statement

The Employee Training Gap

📚

Static Documents

Training materials sit in SharePoint folders. Employees can't query them conversationally — they must read everything to find one answer.

📉

No Progress Visibility

Managers can't see which technologies employees understand. There's no quiz feedback loop to guide upskilling priorities.

🔄

Manual Content Updates

Writing new training modules for every tool version update requires dedicated L&D effort. There's no way to auto-draft content from existing material.

TechTrainer AI solves this with a multi-agent system that answers questions, generates quizzes, recommends learning paths, and drafts content — all grounded in your own training documents.

7 Core Features

Chat Q&A

RAG-powered answers cited from your documents. Persistent history across browser refreshes (ChromaDB, last 50 exchanges).

Multi-choice quizzes with explanations, scored and written to SQLite. Select technology, difficulty, and question count.

Learning Path

Personalized study roadmap from quiz history: current level, recommended topics, weak/strong areas, estimated hours.

AI drafts full training modules in Markdown. Preview, download, or save to library. Sandboxed to ./data/generated/.

Knowledge Base

Upload documents per topic; background indexing with per-topic progress bar and Stop button. Custom topic support.

📊
Progress Dashboard

Sidebar with per-technology score bars, study streak, and session info updated in real time.

⚙️ Settings Tab: Switch AI provider, configure embedding, adjust model parameters — all saved to .env / settings.json without restart.

Pattern: Each sub-agent is a @tool function. The Orchestrator calls it like a tool — Strands instantiates a fresh agent inside, runs it, and returns the result. Sub-agents are topic-agnostic — they retrieve from whichever KB collections exist.

Agent Design

Training Agent — Any Knowledge Base

📚 training_agent
A single topic-agnostic agent that retrieves from any indexed Knowledge Base collection. The topic is determined at runtime from the user's query and the available ChromaDB collections — not hardcoded into the agent.
@tool def training_agent( query: str, topic: str, # resolved from KB registry ) -> str: """Answer questions using any indexed topic. Retrieves from the matching ChromaDB collection and returns cited content."""
AgentSkills — Pluggable Domain Files

Skill files (.md) provide agent-level hints about a topic's conventions and terminology. They are optional and loaded dynamically from the KB registry — not fixed to any built-in list.

topic_a.md topic_b.md topic_c.md topic_d.md your_topic.md … one per KB topic

Safety & Security

Safety & Security Controls

🛡️
Bedrock Guardrails (optional)
  • Content filtering — blocks toxicity, hate speech
  • Off-topic query blocking (education only)
  • PII detection logging (defense-in-depth)
  • Activated via BEDROCK_GUARDRAIL_ID in .env
⚙️
LoggingThrottleHook
  • Max 10 tool calls per turn (prevents runaway loops)
  • All tool calls logged with name, inputs, duration
  • Tool results logged on success/failure
  • BeforeToolCallEvent + AfterToolCallEvent hooks
⏱️
Timeout Safety
  • All sub-agent calls wrapped in 60-second watchdog
  • Timeout triggers agent.cancel() cleanly
  • UI surfaces timeout error to user
  • ModelRetryStrategy — 3 retries on Bedrock throttle
📁
File Write Sandbox
  • Content Author restricted to ./data/generated/
  • Path traversal attempts blocked in file_write tool
  • AWS credentials stored in .env — never committed
  • Secret keys never written to settings.json
