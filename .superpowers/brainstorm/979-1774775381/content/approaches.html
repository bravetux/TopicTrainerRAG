Where should Knowledge Base management live?

Three structural approaches — pick what feels right

A

New "Knowledge Base" tab ★ Recommended

Add a 5th tab 📚 Knowledge Base alongside Chat, Quiz, Learning Path, Content Author.

  • Full-width layout for all 10 technologies
  • Upload widget + status table + index controls per tech
  • Fits the existing tab pattern — no restructuring needed
  • Clear separation from learner-facing tabs
B

Expand the sidebar

Add KB management to the left sidebar (currently shows progress stats only).

  • Always visible without switching tabs
  • But sidebar is narrow — file upload UX suffers
  • Crowds out the progress dashboard already there
C

Separate Streamlit page

Create pages/knowledge_base.py as a dedicated admin page in Streamlit's multi-page app.

  • Clean admin/user separation
  • Requires converting app.py to multi-page structure
  • More refactoring for little extra benefit at this scale
